# План развития приложения

Этот проект продолжает готовый сервис календарного бронирования. План ниже
фиксирует небольшие фичи, баги и quality-задачи для дальнейшей разработки.

## Цель ближайшего этапа

- Перенести рабочее приложение из предыдущего репозитория.
- Зафиксировать список небольших фич, багов и quality-задач.
- Добавить scheduled workflow для регулярной автоматической проверки.

## Предлагаемые issues

### Bug: обработка недоступного API на фронтенде

Сейчас пользовательский сценарий должен явно показывать ошибку, если backend
недоступен или возвращает unexpected response. Агент может добавить понятные
empty/error states и e2e-покрытие.

Ожидаемый результат:

- UI не зависает при ошибке API.
- Пользователь видит понятное сообщение.
- Есть тест на ошибочный ответ или недоступный endpoint.

### Bug: улучшить валидацию формы бронирования

Форма бронирования должна устойчиво обрабатывать пустые поля, некорректный
email и ошибки backend validation.

Ожидаемый результат:

- Поля валидируются до отправки.
- Ошибки backend отображаются рядом с формой.
- Покрыты успешный и неуспешный сценарии.

### Feature: фильтрация слотов по дате

На странице event type можно добавить компактный фильтр по дате, чтобы
пользователь быстрее находил подходящий слот.

Ожидаемый результат:

- Список слотов фильтруется по выбранной дате.
- Сброс фильтра возвращает все доступные слоты.
- Поведение покрыто e2e-тестом.

### Feature: улучшить админский список бронирований

Админская страница бронирований может получить сортировку и визуальные статусы
для более удобной проверки заявок.

Ожидаемый результат:

- Бронирования сортируются по времени.
- В списке явно видны имя, email, тип события и слот.
- Есть тест на отображение seed bookings.

### Quality: scheduled Lighthouse check

Добавить GitHub Actions workflow, который запускается по расписанию и собирает
Lighthouse-отчет для публичного или локально поднятого приложения.

Ожидаемый результат:

- Workflow запускается по `schedule` и вручную через `workflow_dispatch`.
- Отчет сохраняется как artifact.
- В README описано, где смотреть результат.

### Quality: scheduled e2e smoke test

Добавить ночной smoke workflow для проверки базового сценария бронирования.

Ожидаемый результат:

- Workflow запускается по расписанию.
- Проверяется основной user journey.
- Playwright report сохраняется как artifact при падении.
